# Test Strategy – SauceDemo Testing Project

## Objective

The objective of this testing project is to validate core functionalities of the SauceDemo e-commerce application using manual testing techniques.

---

## Scope of testing

The following functionalities were tested:

- User login
- Login validation
- Adding products to the cart
- Removing products from the cart
- Cart counter behavior
- Checkout process
- Price calculation
- Reset App State functionality

---

## Types of testing performed

During the testing process the following testing techniques were applied:

- Functional Testing
- Manual Testing
- Exploratory Testing
- Negative Testing
- UI State Validation
- Navigation Testing

---

## Test artifacts produced

The following testing artifacts were created during this project:

- Test Cases
- Bug Reports
- Exploratory Testing Notes
- Test Evidence (Screenshots)

---

## Testing approach

The application was tested using a user-flow approach, starting from login and continuing through the complete purchase process.

The goal was to validate the behavior of the system under normal and edge case scenarios.
